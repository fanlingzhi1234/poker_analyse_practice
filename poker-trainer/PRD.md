# Poker Trainer PRD (MVP v0.1)

## 1. 产品一句话
一个面向初中级德州扑克玩家的 **局面训练 Web App**：输入手牌、公牌、玩家人数和对手范围，输出 **胜率、成牌概率、粗粒度行动建议和教学解释**。

---

## 2. 目标与非目标

### 2.1 目标
- 帮用户建立对 **牌力 / 胜率 / 对手范围** 的基础直觉（MVP 聚焦单对手训练）
- 支持单手复盘和情景训练
- 让结果不只是数字，而是可解释的学习反馈

### 2.2 非目标
- 不做职业级 GTO / solver
- 不做复杂下注树、SPR、ICM、锦标赛模型
- 不做实时实战外挂定位
- 不做社交、排行榜、账号体系优先版本

---

## 3. 目标用户

### 核心用户
1. 初中级德州玩家
2. 想系统训练“范围感”和“牌感”的用户
3. 经常玩 heads-up / 6-max / 9 人桌的现金局或娱乐局玩家

### 非核心用户
- 只想娱乐、不愿输入局面的轻度用户
- 职业级玩家（会嫌模型过粗）

---

## 4. 核心场景

### 场景 A：单手复盘
输入一手真实或假设局面，查看：
- 当前胜率
- 当前牌力和听牌情况
- 对手范围假设下的行动建议
- 为什么会得到这个建议

### 场景 B：对比训练
对同一局面切换：
- 玩家人数：HU / 6-max / 9-max
- 对手范围：any two / loose / standard / tight / premium

目标：理解人数和范围如何改变 equity 与决策。

### 场景 C：快速学习
用户不知道某类牌在某个牌面到底强不强，通过工具获得直观反馈。

---

## 5. MVP 功能范围

## 5.1 输入功能
- Hero 手牌输入：2 张
- 公牌输入：0~5 张
- 玩家人数选择：2 / 6 / 9
- 对手范围选择：
  - any two
  - loose
  - standard
  - tight
  - premium
- 合法性校验：
  - 重复牌检测
  - 缺牌/非法输入阻止分析

## 5.2 输出功能
- 胜率 / 平率 / 败率
- 当前牌力识别：
  - 高牌 / 一对 / 两对 / trips / 顺子 / 同花 / 葫芦 / 四条
- 听牌识别：
  - flush draw / OESD / gutshot / combo draw
- 成牌概率（未来街）
- 粗粒度行动建议：
  - fold / cautious call / call / raise
- 教学解释：
  - 当前局面的核心逻辑
  - 多人池影响
  - 对手范围对结果的影响

## 5.3 训练增强功能（MVP 可做轻量版）
- 一键切换人数重算
- 一键切换范围重算
- 输出一句“记忆点总结”

---

## 6. 明确不做（v1）
- 高级 13x13 range editor
- 多个对手分别设定独立范围
- 下注树和复杂街道动作历史
- Solver/GTO/Nash 级最优策略
- 历史手牌库/社区功能/登录体系

---

## 7. UX 流程

### 7.1 单次训练流程
1. 选择桌型：HU / 6-max / 9-max
2. 输入 Hero 手牌
3. 输入公共牌
4. 选择对手范围预设
5. 点击“分析”
6. 查看结果：
   - 胜率
   - 当前牌力
   - 听牌情况
   - 行动建议
   - 原因解释
7. 切换范围或人数，做对比训练

### 7.2 UX 原则
- 输入尽量快，不搞复杂配置
- 结果页“结论先行，解释随后”
- 教学解释用白话，不堆术语
- 明确提示：建议为学习型 heuristic，不是最优实战结论

---

## 8. 页面信息架构

### 页面 1：输入页
- 桌型选择
- 手牌输入区
- 公牌输入区
- 范围选择区
- 分析按钮

### 页面 2：结果页
- 顶部结果卡：胜率 / 平率 / 败率
- 当前牌力卡
- 听牌与成牌概率卡
- 行动建议卡
- 解释卡（为什么）
- 对比训练快捷入口

### 页面 3：对比训练页（可选，若时间不够可合并到结果页）
- 同一局面下切换人数与范围
- 对比不同结果

---

## 9. 成功标准（MVP）
- 用户可在 30 秒内完成一手局面输入并看到结果
- 基础胜率计算稳定正确
- 教学解释能回答“为什么不是强牌/为什么只能谨慎跟注”
- 至少支持 3 类常见训练维度：
  - 人数变化
  - 范围变化
  - 公牌变化

---

## 10. 风险与约束

### 风险 1：建议动作看起来过度权威
解决：文案统一标注为“学习建议 / heuristic”。

### 风险 2：多人池模型过于简化
解决：v1 文案写清楚这是“典型对抗环境假设”。

### 风险 3：范围预设不透明导致不信任
解决：为每个预设范围提供文字说明和代表性手牌例子。

### 风险 4：结果只给数字，学习价值低
解决：把“解释层”和“对比训练”作为 MVP 的一部分，而不是锦上添花。

---

## 11. 当前建议结论
先做成一个 **轻量、可解释、对比训练友好** 的 Web App。
第一版重点是：
1. 正确计算
2. 范围切换清晰
3. 解释有教学价值
4. 不碰过重的专业求解

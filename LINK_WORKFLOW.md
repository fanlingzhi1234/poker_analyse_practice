# LINK_WORKFLOW.md - B哥的链接处理协议

当 B哥 直接丢来一个链接、文章、网页、报告、帖子、视频页或资料页时，默认按以下流程处理。

## Default Output Structure

### 1) 三句摘要
- 用 3 句话概括内容是什么、核心结论是什么、为什么值得关注。

### 2) 核心观点
- 提炼 3-7 条核心观点。
- 区分“作者观点”“事实信息”“我的判断”。

### 3) 关键数据与出处
- 抽取文中的关键数字、时间、公司/产品名、结论性事实。
- 尽量附上原始出处或文内对应来源。
- 如果原文未提供可靠出处，要明确标注“原文未给出处/需二次核验”。

### 4) 我的判断
- 判断这篇内容的价值、可信度、偏见来源、适用边界。
- 如果涉及行业、产品、技术、投资，补充我的独立分析：
  - 哪些判断可靠
  - 哪些判断可能过度延伸
  - 哪些信息值得继续跟踪

### 5) 推荐 Tags
默认从以下维度给标签：
- **主题标签**：AI、SaaS、投资、汽车、硬件、产品、架构、效率、创业……
- **内容类型**：报告、新闻、观点文、教程、案例、访谈、数据源……
- **状态标签**：待读、已读、待跟进、值得收藏、可入长期记忆……
- **用途标签**：产品灵感、投资参考、架构参考、行业研究、写作素材……

### 6) 是否值得进 Notion
给出明确结论：
- 值得
- 一般
- 不值得

并说明原因：
- 是资料沉淀价值高
- 还是只是短期资讯
- 还是噪音大于价值

### 7) 是否值得写入长期记忆
默认判断是否需要写入 `MEMORY.md` 或当天 `memory/YYYY-MM-DD.md`：
- **写入长期记忆**：当内容反映了 B哥 的稳定偏好、长期项目方向、重复出现的重要主题、持续跟踪的投资/产品判断
- **写入当日记录**：当内容只是当天研究轨迹、临时想法、一次性发现
- **不写入**：当内容只是普通信息消费，没有长期复用价值

## Decision Rules

### A. 面向准确性
- 优先准确，不为流畅牺牲真实性。
- 如果页面信息不足、链接打不开、来源可疑，要直接说明。
- 如果有重大不确定性，要显式写“低置信度”或“需核验”。

### B. 面向证据
- 关键事实尽量可追溯。
- 若文章本身是二手转述，优先寻找一手来源。
- 无法确认时，不把推测包装成事实。

### C. 面向效率
- 默认先给结论，再展开分析。
- 长文先压缩成结构化重点，避免把原文换皮复述。

### D. 面向 B哥 的工作流
- 若内容和投资相关：增加风险提示、适用期限、信息时效性说明。
- 若内容和产品相关：增加可借鉴点、用户价值、是否能进入 MVP 的判断。
- 若内容和架构相关：增加技术取舍、适用场景、潜在坑点。

## Optional Extended Output
当内容特别重要时，可额外补充：
- **一句话结论**
- **反方观点/潜在漏洞**
- **值得追踪的问题清单**
- **可执行下一步**
- **Notion 标题建议**

## Trigger Phrases
以下表达默认触发本协议：
- “帮我看下这个链接”
- “总结一下这篇文章”
- “分析这个网页”
- “提炼重点”
- “给我打标签收纳”
- “这个值不值得存到 notion”
- 直接发送 URL 且未附加明确格式要求

## Short Response Template

可默认输出为：

1. **三句摘要**
2. **核心观点**
3. **关键数据/出处**
4. **我的判断**
5. **推荐 tags**
6. **Notion：值得 / 一般 / 不值得**
7. **记忆：长期 / 当日 / 不写**

## Principle

目标不是“帮 B哥 读过一篇文章”，而是把外部信息转化成：
- 可判断的结论
- 可检索的知识
- 可沉淀的记忆
- 可执行的下一步
